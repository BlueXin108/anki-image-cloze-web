# Vercel Analytics 接入说明

## 当前已完成

当前前端已经接入 `@vercel/analytics` 的基础统计能力。

这次先接的是最基础、最稳的一层：

- 自动页面访问统计
- 路径级访问数据
- 来源、设备、地区等基础访问画像

当前还没有默认增加一堆业务事件埋点，例如“点了哪个按钮”“导出了几次”。这样做是为了先把基础盘子搭稳，避免一上来就把口径做复杂。

## 代码位置

- 依赖安装：
  - `frontend/package.json`
- 入口挂载：
  - `frontend/src/main.tsx`

## 现在能看到什么

只要站点部署在 Vercel 上，并且有真实访问，后续就能在 Vercel 后台看到这类基础数据：

- 页面访问量
- 访问路径
- 访问来源
- 设备类型
- 浏览器
- 国家 / 地区等基础分布

如果后面你想继续往前走，还可以再补：

- 自定义事件
  - 例如“开始导出”“生成 APKG”“打开 Anki 帮助”
- 性能指标
  - 这部分通常再配合 `@vercel/speed-insights`

## 去哪里看

部署到 Vercel 后，打开对应项目后台，一般可以在：

- `Analytics`

里看到页面访问相关统计。

首次使用时，记得先到项目后台把它打开：

1. 进入对应的 Vercel 项目
2. 打开左侧或顶部的 `Analytics`
3. 点击启用

如果后面你再接入性能监测，则通常还会多一个：

- `Speed Insights`

## 使用前提

- 站点需要部署在 Vercel 上
- 需要有真实访问数据进入
- 本地开发环境可以保留这段代码，但统计价值主要体现在部署后的站点

## 当前建议

先观察一段时间基础访问数据，再决定要不要加业务事件。比较值得补的下一批事件通常是：

- 导入图片
- 打开导出流程
- 导出到 Anki
- 导出 APKG
- 导出纯图像组
- 打开 Anki 帮助

这样后面你就能知道：

- 用户到底更常用哪种导出方式
- AnkiConnect 直连使用率高不高
- 有多少人只走 APKG
- 帮助说明是否经常被打开
