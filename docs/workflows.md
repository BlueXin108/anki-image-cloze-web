# 工作流说明

## 用户工作流：流水线模式

1. 选择目录
2. 运行第一阶段
3. 确认归档位置
4. 运行第二阶段
5. 确认遮罩结果
6. 导出

## 用户工作流：手动模式

1. 切换到手动模式
2. 选择图片
3. 调整裁切与遮罩
4. 需要时合并多个遮罩块为同一张卡
5. 预览最终结果
6. 导出

## 维护者工作流

当要做中等以上规模改动时：

1. 先判断改动是只影响流水线模式、只影响手动模式，还是两者共享。
2. 如果是共享改动，两个模式都要验证。
3. 如果改动涉及导出行为，同时验证：
   - 预览
   - 最终导出逻辑
   - Anki 模板行为
4. 条件允许时，在同一任务内同步更新文档。

## 文档维护

以下场景需要更新对应文档：

- 控制规则或文档索引变化
  - 更新 `.agent`
- 产品范围或术语变化
  - 更新 `docs/project-overview.md`
- 职责边界变化
  - 更新 `docs/architecture.md`
- 出现真实踩坑、误判、修正经验
  - 更新 `docs/pitfalls-and-corrections.md`
- 出现环境层、协作层、工具层经验
  - 更新 `docs/agent-experience.md`
