# 工作流说明

## 用户工作流：手动模式

1. 上传图片或导入文件夹
2. 如有需要，恢复浏览器上次保存的项目
3. 调整裁切与遮罩
4. 需要时合并多个遮罩块为同一张卡
5. 切换不同卡片分组查看即时预览
6. 通过右下角导出入口打开全屏导出流程
7. 按顺序逐张确认当前图片的牌组与标签
8. 在最后一步调整导出质量并统一导出到本机 Anki

## 用户工作流：流水线占位

1. 切换到流水线占位
2. 查看当前保留的 UI 位置
3. 等后续把自动能力逐步接回

## 维护者工作流

当要做中等以上规模改动时：

1. 先判断改动是只影响手动网页链路，还是会影响后续保留的流水线占位。
2. 涉及共享编辑器时，同时验证普通编辑和聚焦编辑。
3. 涉及导入行为时，同时验证：
   - 网页预览
   - 浏览器本地保存
   - AnkiConnect 直连
   - 全屏导出流程里的逐张确认状态
4. 条件允许时，在同一任务内同步更新文档。

## 文档维护

以下场景需要更新对应文档：

- 控制规则或文档索引变化
  - 更新 `.agent`
- 产品范围或术语变化
  - 更新 `docs/project-overview.md`
- 职责边界变化
  - 更新 `docs/architecture.md`
- 出现真实踩坑、误判、修正经验
  - 更新 `docs/pitfalls-and-corrections.md`
- 出现环境层、协作层、工具层经验
  - 更新 `docs/agent-experience.md`
