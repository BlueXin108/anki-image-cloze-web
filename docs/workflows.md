# 工作流说明

## 用户工作流：手动模式

1. 上传图片或导入文件夹
2. 导入时自动做压缩和最大分辨率校正
3. 若检测到上次本地项目，右上角会弹出恢复提示，但默认不自动恢复
4. 如有需要，再由用户手动恢复浏览器上次保存的项目
5. 调整裁切与遮罩
6. 需要时合并多个遮罩块为同一张卡
7. 切换不同卡片分组查看即时预览
8. 通过右下角导出入口打开全屏导出流程
9. 如有需要，可先在设置里切换制卡模式
10. 按顺序逐张确认当前图片的牌组与标签
11. 在最后一步选择导出路径
12. 桌面端可统一导出到本机 Anki、下载 APKG，或生成纯图像组
13. 若当前仍持有原图链路，可在三条导出路径里统一选择 `WebP / JPG / PNG`
14. 若导入时已经压缩，或中途执行过项目压缩，导出页会自动锁住 `PNG / JPG` 并说明原因
15. 若当前制卡模式是“整张图一张卡，点遮罩切换”，则 `AnkiConnect / APKG` 会保留点击切换；纯图像组自动退成静态前后图
16. 移动端可生成 APKG 或纯图像组，并尽量走系统分享
17. 若桌面端直连失败，可从右上角帮助继续完成 AnkiConnect 配置
18. 若浏览器支持，也可以把网页安装到桌面或主屏，方便以后像应用一样打开

## 用户工作流：手动模式（移动端）

1. 上传图片或导入文件夹
2. 在竖排图片区里切换当前图片
3. 常规页只做预览
4. 进入聚焦编辑后完成遮罩编辑与切图
5. 关闭聚焦编辑后，下次打开会重新创建一份新的聚焦编辑实例
6. 若手机端本次编辑过程中出现局部交互异常，可优先通过“关掉再打开聚焦编辑”做局部复位，而不必先刷新整页
7. 逐张确认牌组与标签
8. 生成 APKG 或纯图像组，并在支持时直接交给系统分享
9. 若用户回到电脑端，同一条链路可以继续接入 AnkiConnect 直连
10. 若浏览器支持，也可以把网页添加到主屏，但离线时只保证应用壳和静态界面可打开

## 用户工作流：流水线占位

1. 切换到流水线占位
2. 查看当前保留的 UI 位置
3. 等后续把自动能力逐步接回

## 维护者工作流

当要做中等以上规模改动时：

1. 先判断改动是只影响手动网页链路，还是会影响后续保留的流水线占位。
2. 涉及共享编辑器时，同时验证普通编辑和聚焦编辑。
3. 涉及导入行为时，同时验证：
  - 网页预览
  - 浏览器本地保存
  - 导入预处理后的尺寸与清晰度
  - AnkiConnect 直连
  - APKG 导出
  - 纯图像组导出
  - 制卡模式切换后，卡片数量是否同步变化
  - 全屏导出流程里的逐张确认状态
  - PNG / JPG 是否只在原图链路下开放
  - 中途项目压缩后，导出格式是否立即重新收窄
4. 若改动涉及移动端，还要同时回看：
   - 常规页是否仍然保持只读
   - 聚焦编辑是否仍然能完成真正编辑
   - 聚焦页左右切图是否还可用
   - 手机端关闭再打开聚焦编辑时，是否仍然会拿到一份新的编辑实例
5. 条件允许时，在同一任务内同步更新文档。
6. 若改动涉及 PWA，还要同时回看：
   - 安装清单是否仍然有效
   - 离线缓存是否只覆盖网页壳和静态资源
   - 是否误把项目数据或 Anki 连接请求交给 Service Worker

## 用户帮助流

当用户不知道该走哪条导出路径时：

1. 先看顶部当前端模式提示
2. 若只是想把卡包导出去，直接走 APKG
3. 若只是想拿到一组最大兼容的前后图，直接走纯图像组
4. 若想读取本机牌组或一键写入本机 Anki：
   - 桌面端点右上角 `Anki 帮助`
   - 或在首次直连失败时，从右上角消息提示进入帮助
5. 根据帮助里的示意图完成：
   - 安装 Anki
   - 安装 AnkiConnect
   - 在插件设置里放行当前站点

## 当前状态栏行为

- 左下状态胶囊默认不主动展开
- 只有真正出错时，才会自动展开失败项
- 当前常驻跟踪的重点任务是：
  - Anki 连接
  - 项目恢复
  - 浏览器本地保存

## 文档维护

以下场景需要更新对应文档：

- 控制规则或文档索引变化
  - 更新 `.agent`
- 产品范围或术语变化
  - 更新 `docs/project-overview.md`
- 职责边界变化
  - 更新 `docs/architecture.md`
- 性能缓存策略或导出开放规则变化
  - 更新 `docs/performance-and-export-policy.md`
- 出现真实踩坑、误判、修正经验
  - 更新 `docs/pitfalls-and-corrections.md`
- 出现环境层、协作层、工具层经验
  - 更新 `docs/agent-experience.md`
