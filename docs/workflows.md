# 工作流说明

## 用户工作流：手动模式

1. 上传图片或导入文件夹
2. 导入时自动做压缩和最大分辨率校正
3. 如有需要，恢复浏览器上次保存的项目
4. 调整裁切与遮罩
5. 需要时合并多个遮罩块为同一张卡
6. 切换不同卡片分组查看即时预览
7. 通过右下角导出入口打开全屏导出流程
8. 按顺序逐张确认当前图片的牌组与标签
9. 在最后一步选择导出路径
10. 桌面端可统一导出到本机 Anki、下载 APKG，或生成纯图像组
11. 移动端可生成 APKG 或纯图像组，并尽量走系统分享
12. 若桌面端直连失败，可从右上角帮助继续完成 AnkiConnect 配置

## 用户工作流：手动模式（移动端）

1. 上传图片或导入文件夹
2. 在竖排图片区里切换当前图片
3. 常规页只做预览
4. 进入聚焦编辑后完成遮罩编辑与切图
5. 逐张确认牌组与标签
6. 生成 APKG 或纯图像组，并在支持时直接交给系统分享
7. 若用户回到电脑端，同一条链路可以继续接入 AnkiConnect 直连

## 用户工作流：流水线占位

1. 切换到流水线占位
2. 查看当前保留的 UI 位置
3. 等后续把自动能力逐步接回

## 维护者工作流

当要做中等以上规模改动时：

1. 先判断改动是只影响手动网页链路，还是会影响后续保留的流水线占位。
2. 涉及共享编辑器时，同时验证普通编辑和聚焦编辑。
3. 涉及导入行为时，同时验证：
   - 网页预览
   - 浏览器本地保存
   - 导入预处理后的尺寸与清晰度
   - AnkiConnect 直连
   - APKG 导出
   - 纯图像组导出
   - 全屏导出流程里的逐张确认状态
4. 若改动涉及移动端，还要同时回看：
   - 常规页是否仍然保持只读
   - 聚焦编辑是否仍然能完成真正编辑
   - 聚焦页左右切图是否还可用
5. 条件允许时，在同一任务内同步更新文档。

## 用户帮助流

当用户不知道该走哪条导出路径时：

1. 先看顶部当前端模式提示
2. 若只是想把卡包导出去，直接走 APKG
3. 若只是想拿到一组最大兼容的前后图，直接走纯图像组
4. 若想读取本机牌组或一键写入本机 Anki：
   - 桌面端点右上角 `Anki 帮助`
   - 或在首次直连失败时，从右上角消息提示进入帮助
5. 根据帮助里的示意图完成：
   - 安装 Anki
   - 安装 AnkiConnect
   - 在插件设置里放行当前站点

## 当前状态栏行为

- 左下状态胶囊默认不主动展开
- 只有真正出错时，才会自动展开失败项
- 当前常驻跟踪的重点任务是：
  - Anki 连接
  - 项目恢复
  - 浏览器本地保存

以下场景需要更新对应文档：

- 控制规则或文档索引变化
  - 更新 `.agent`
- 产品范围或术语变化
  - 更新 `docs/project-overview.md`
- 职责边界变化
  - 更新 `docs/architecture.md`
- 出现真实踩坑、误判、修正经验
  - 更新 `docs/pitfalls-and-corrections.md`
- 出现环境层、协作层、工具层经验
  - 更新 `docs/agent-experience.md`
