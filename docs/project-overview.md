# 项目总览

## 这个项目是什么

这个项目当前是一条面向 Web / Vercel 的 Anki 图像挖空工作台分线。它的目标是让用户在网页里直接完成图片导入、手动遮挡、即时预览、逐张确认牌组，并最终走两条导出路径之一：

- `桌面端`
  - 可选择通过 AnkiConnect 直连到本机 Anki
- `全端`
  - 都可以生成 APKG 卡包

## 主要使用路径

### 手动模式

当前的主使用路径，适合希望直接控制图片、裁切和遮罩的场景。

典型流程：

1. 上传图片或导入文件夹
2. 调整裁切范围
3. 手动画遮罩或编辑遮罩
4. 需要时把多个遮罩块合并成同一张卡
5. 通过右下角导出入口进入全屏导出流程
6. 逐张确认牌组与标签，并查看只读预览
7. 在最后一步统一调整导出质量
8. 桌面端可直连导入本机 Anki
9. 移动端优先生成 APKG 卡包，并复用当前设备的本地牌组池
10. 如果用户不想配置 AnkiConnect，可以始终只走 APKG

### 手动模式：移动端

当前移动端路线已经单独收口：

1. 上传图片或导入文件夹
2. 在竖排图片区里选图
3. 常规页只做预览，避免误触
4. 进入聚焦编辑后再画遮罩、改遮罩、切图
5. 逐张确认牌组与标签
6. 生成 APKG 卡包，并优先尝试系统分享
7. 若用户想直连本机牌组与发卡，顶部会提示改到电脑端继续

### 流水线模式

当前保留为 UI 占位，用来给后续 OCR、自动归类或自动遮罩能力留接口位置。

## 核心概念

- `draft`
  - 一张图片在工作台中的当前工作状态
- `mask`
  - 一个遮罩矩形块
- `card group`
  - 一个或多个遮罩块组成的一张最终卡片
- `deck`
  - 当前图片准备导入到哪个 Anki 牌组

## 当前范围

当前这版网页工作台已经覆盖：

- 多图上传
- 文件夹导入
- 浏览器本地保存
- 本地牌组池
- 手动编辑
- 移动端竖排布局
- 移动端聚焦编辑
- 即时预览
- 全屏逐张确认导出流
- 右上角 Anki 帮助
- 顶部桌面 / 手机端模式提示
- 文本里的轻量强调提示块
- 首次 Anki 连接失败时的轻提示入口
- 直接连接 AnkiConnect 导入
- APKG 导出

## 后续方向

后续方向是：

- 在不破坏当前手动网页链路的前提下
- 逐步把 OCR、自动归类、自动遮罩的一部分能力接回网页版本
