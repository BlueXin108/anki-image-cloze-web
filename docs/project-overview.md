# 项目总览

## 这个项目是什么

这个项目当前是一条面向 Web / Vercel 的 Anki 图像挖空工作台分线。它的目标是让用户在网页里直接完成图片导入、手动遮挡、即时预览、逐张确认牌组，并最终走三条导出路径之一：

补充说明：

- 当前网页版本基于前端本地处理
- 图片导入、裁切、遮罩、预览、导出和浏览器本地保存都在用户当前设备完成
- 不依赖把图片上传到项目服务端再处理

- `桌面端`
  - 可选择通过 AnkiConnect 直连到本机 Anki
- `全端`
  - 都可以生成 APKG 卡包
  - 都可以生成纯图像组压缩包

## 主要使用路径

### 手动模式

当前的主使用路径，适合希望直接控制图片、裁切和遮罩的场景。

典型流程：

1. 上传图片或导入文件夹
2. 可按需决定是否在导入时先做压缩和最大分辨率校正
3. 若浏览器里已有上次项目，右上角会提示可恢复，但默认不自动恢复
4. 调整裁切范围
5. 手动画遮罩或编辑遮罩
6. 需要时把多个遮罩块合并成同一张卡
7. 通过右下角导出入口进入全屏导出流程
8. 逐张确认牌组与标签，并查看只读预览
9. 在最后一步选择导出路径
10. 如有需要，可在设置里先切换制卡模式
11. 桌面端可直连导入本机 Anki
12. 全端都可生成 APKG 或纯图像组
13. 如果当前仍是原图链路，三条导出路径都可统一选择 `WebP / JPG / PNG`
14. 如果导入时已压缩，或中途执行过项目压缩，导出页会自动锁住不再合规的格式
15. 如果用户不想配置 AnkiConnect，可以始终只走 APKG 或纯图像组

### 手动模式：移动端

当前移动端路线已经单独收口：

1. 上传图片或导入文件夹
2. 在竖排图片区里选图
3. 常规页只做预览，避免误触
4. 进入聚焦编辑后再画遮罩、改遮罩、切图
5. 若中途觉得这层状态不对，直接关闭再打开即可拿到一份新的聚焦编辑实例
6. 聚焦编辑关闭后，下次打开会重新创建，避免把上一次残留的临时状态带进来
7. 逐张确认牌组与标签
8. 生成 APKG 或纯图像组，并在支持时优先尝试系统分享
9. 若用户想直连本机牌组与发卡，顶部会提示改到电脑端继续

### 流水线模式

当前保留为 UI 占位，用来给后续 OCR、自动归类或自动遮罩能力留接口位置。

## 核心概念

- `draft`
  - 一张图片在工作台中的当前工作状态
- `mask`
  - 一个遮罩矩形块
- `card group`
  - 一个或多个遮罩块组成的一张最终卡片
- `card generation mode`
  - 决定一组遮罩最后要生成多少张卡，以及题面 / 答案面默认怎么显示
- `deck`
  - 当前图片准备导入到哪个 Anki 牌组

- 多图上传
- 文件夹导入
- 浏览器本地保存

- 手动编辑
- 移动端竖排布局
- 移动端聚焦编辑
- 移动端聚焦编辑关闭后自动重建实例
- 即时预览
- 全屏逐张确认导出流
- 右上角 Anki 帮助
- 顶部设置入口
- 顶部桌面 / 手机端模式提示
- 文本里的轻量强调提示块

- 纯图像组导出

- 预览缓存与导出格式联动规则

- 导出完成后的收尾页与项目清理确认

## 最近值得关注的变化

当前版本有几条对用户最有感知的调整：

- 首页与工作台之间的切换被重新整理，首屏不再只是静态入口，而是更像一个完整开场。
- 导出完成后不再额外跳一个完全独立的提示框，而是在原导出窗口里直接进入完成页。
- 移动端入口被重新收口，上传、拍摄和文件管理器的职责更清楚。
- 关闭动画后，首屏背景会直接停止渲染，避免进入工作台后还在后台保留不必要的动效开销。

## 后续方向

后续方向是：

- 在不破坏当前手动网页链路的前提下
- 逐步把 OCR、自动归类、自动遮罩的一部分能力接回网页版本
