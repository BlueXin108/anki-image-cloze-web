# 更新记录

## 0.9.6

发布时间：2026-04-08

### 这次更新了什么

- 补回了一批在主文件回退后被连带冲掉的手机端和导入体验细节，页面表现重新和后续版本保持一致。
- 手机底部说明恢复成低调的勾选式提示，同时重新给底部导出按钮留出空间，不再互相遮挡。
- `HEIF / HEIC` 导入链路重新理顺：会先提醒这类图片转换更慢，普通图片优先进项目，`HEIF` 先以“转换中”占位挂到图片列表里，转完后再开放编辑。
- `HEIF` 和普通压缩图片的后缀显示重新统一，列表里会按最终结果显示，不再只修一半。
- 手机端“拍摄”入口重新接回系统相机，并按浏览器类型做更保守的显示策略，避免在不稳定浏览器里误导用户。
- 电脑端拖入整个文件夹时，重新支持继续向下读取里面的图片，不再只把文件夹本身当成空对象。
- `APKG` 导出重新收回到更稳的方式：不再额外弹新页面，也不再在手机端顺手再打开一次文件，减少下载命名和接收过程里的混乱。
- 修了一处会把页面拖进反复更新的公共引用问题，避免按钮、滚动区这类位置再次把页面带炸。

### 升级提醒

- 这次是本地小版本整理，重点是把之前已经做过、但被回退影响的体验补齐。
- 如果浏览器还挂着旧页面缓存，刷新一次后再试导入和导出，体验会更接近当前版本。

## 0.9.5

发布时间：2026-04-07

### 这次更新了什么

- 全面梳理了前端分包与缓存策略，首屏更轻，工具页和高频入口的加载更稳定。
- PWA 现在更像真正的工具应用，网页代码和静态资源会更积极地缓存，本地再次打开时不必反复重下整套前端。
- 启动时不再默认恢复上次项目，改成右上角提示后由用户自己决定是否恢复。
- 图片处理、导出和浏览器本地保存的说明统一收口，明确告诉用户整个过程都在当前设备完成，不上传图片，减少流量和隐私顾虑。
- 手动编辑区与聚焦编辑区的快捷键做了统一，切图、删除、合并卡片等操作更顺手。
- 图片列表、导出页、分组预览、悬浮按钮、提示层级、补全交互等界面细节做了一轮整理，减少遮挡、撑开和误触。
- 导出流程统一了 `AnkiConnect / APKG / 纯图像组` 的图片格式规则。
- 现在只有明确仍保留原图链路时，才开放 `WebP / JPG / PNG`；如果导入时压缩过，或中途执行过项目压缩，会自动锁住不再合适的格式并说明原因。
- 下方问题面 / 答案面预览做了缓存和减震处理，减少重复重绘、闪烁和页面抽动。
- 浏览器本地保存从“整批一起重写”改成更轻的方式，单图编辑时卡顿明显降低。
- 新增“慢保存提醒”和项目压缩次数记录，能在编辑明显变慢时给出更温和、更可解释的提示。
- 移动端工作区进一步收口，常规页默认只做预览，真正编辑集中在聚焦编辑层。
- 手机端关闭聚焦编辑后，下次重新打开会重建这一层的临时状态，尽量避免出现必须刷新整个页面才能恢复的情况。
- 项目文档完成了一轮系统更新，当前版本的行为、边界和维护规则已经统一写清楚。

### 升级提醒

- 这次更新里包含 PWA 缓存策略调整。
- 已经打开旧版本网页的用户，通常刷新一次后就能切到新版本。
- 如果浏览器里保存的是旧版本项目记录，导出格式会继续按保守规则处理；需要 `PNG / JPG` 时，建议重新导入原图。
